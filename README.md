# 🗄️ Database Logic Portfolio (SQL)

Bem-vindo ao meu repositório de estratégias e soluções em SQL. Este espaço é dedicado à demonstração de habilidades em modelagem de dados, escrita de queries estruturadas e resolução de problemas reais em bancos de dados relacionais.

## 🚀 Sobre o Repositório

Aqui apresento uma progressão de desafios acadêmicos e práticos, desde a modelagem de entidades únicas até lógicas complexas de relacionamento. O foco principal é demonstrar a capacidade de transformar requisitos de negócio (Histórias de Usuário) em esquemas de banco de dados eficientes e seguros.

### 🛠️ Tecnologias e Ferramentas
* **SGBDs:** PostgreSQL / MySQL.
* **Linguagem:** SQL (DDL, DML, DQL).
* **Modelagem:** Diagramas Entidade-Relacionamento (DER) e Esquemas Lógicos.

## 📂 Projetos e Exercícios

### 1. [Atividade 01: Oficina Mecânica GynAuto](./Atividade%2001:%20Oficina%20Mecânica%20GynAuto)
* **Foco:** Transição de fichas de papel para um sistema digital seguro.
* **Técnicas:** Criação de tabelas (DDL), uso de `PRIMARY KEY` e restrições `UNIQUE` para múltiplos contatos.

### 2. [Atividade 02: Universidade 2C](./Atividade%2002:%20Universidade%202C)
* **Foco:** Padronização de dados docentes com regras de negócio rígidas.
* **Técnicas:** Implementação de tipos enumerados (`ENUM`), validação de domínio com `CHECK` e tratamento de dados opcionais.

### 3. [Atividade 03: Liberty Media - Fórmula 1](./Atividade%2003:%20Liberty%20Media%20-%20Fórmula%201)
* **Foco:** Gestão dinâmica de pilotos, equipes e motores para a temporada 2026.
* **Técnicas:** Operações completas de CRUD, manutenção de registros via `UPDATE` e remoção estruturada com `DELETE`.

### 4. [Atividade 04: Banco SeuDinheiroEMeuGYN](./Atividade%2004:%20Banco%20SeuDinheiroEMeuGYN)
* **Foco:** Implementação de relacionamento 1:1 entre Clientes e Contas.
* **Técnicas:** Uso estratégico de `FOREIGN KEY` combinada com `UNIQUE` para garantir a regra de conta única por CPF.

## 📈 Habilidades Demonstradas
- [x] **DDL:** Criação e gerenciamento de tabelas e tipos customizados.
- [x] **DML:** Manipulação, atualização e exclusão de dados em larga escala.
- [x] **Integridade:** Aplicação de chaves primárias, estrangeiras e restrições de unicidade.
- [x] **Modelagem:** Tradução de requisitos de negócio para Modelos Conceitual e Lógico.
